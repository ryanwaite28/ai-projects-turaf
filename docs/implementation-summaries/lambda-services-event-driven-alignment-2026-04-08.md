## Architecture Principles Enforced

### 1. Event Ownership
- **Principle**: Only one service should own/publish a specific event type
- **Implementation**: 
  - Experiment Service owns `ExperimentCompleted` event
  - Reporting Lambda owns `ReportGenerated` event
  - BFF API does NOT create reports (would violate ownership)

### 2. Command-Query Separation
- **Commands** (writes): Handled by event-driven Lambda functions
- **Queries** (reads): Exposed via BFF API REST endpoints
- Reports are created via events (command), queried via REST (query)

### 3. Asynchronous Processing
- Report generation is inherently async (can take seconds/minutes)
- Tests use polling with `waitHelper.waitForCondition()` to verify completion
- Demonstrates proper handling of eventual consistency

## Event Flow Diagram

```
User Action (Frontend)
    ↓
BFF API: POST /api/v1/experiments/{id}/complete
    ↓
Experiment Service: Publishes ExperimentCompleted event
    ↓
EventBridge: Routes event to subscribers
    ↓
Reporting Lambda: Triggered by event
    ↓
Reporting Lambda: Generates PDF report
    ↓
S3: Stores report file
    ↓
Reporting Lambda: Publishes ReportGenerated event
    ↓
Notification Lambda: Sends notification (future)
    ↓
User: Queries report via BFF API: GET /api/v1/reports
```
